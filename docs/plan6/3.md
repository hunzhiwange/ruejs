# 清单 3：删除 runtime-vapor core / types 里的 `VNode / Child / VNodeType` 契约

在 mounted snapshot 回退链拆掉之后，runtime-vapor 里剩下最显眼的虚拟 DOM 实体就是 `VNode / Child / VNodeType` 本身。只要 `types.rs` 仍定义这些类型，`core.rs` 仍用 `create_element()` 构造 compat `VNode`，就不能说 runtime-vapor 已经真正离开虚拟 DOM 结构。

## 目标

1. 删除 runtime-vapor 中的 `VNode / Child / VNodeType` 结构定义，或者把默认 core 完全迁到非 VDOM 类型后再删除。
2. 移除 `runtime/core.rs` 中返回 `VNode` 的 compat `create_element()` 工厂。
3. 把 `types.rs` 中 `from_vnode_type()` / `into_vnode_type()` 一类命名与转换也一起收口到 `MountInputType` / mounted state 语义。

## 当前阻塞

1. `packages/runtime-vapor/src/runtime/types.rs` 顶部仍直接声明 `VNodeType`、`Child`、`VNode`，并在注释中明确写出“这些类型仍保留在这里”。
2. 同一文件仍使用 `from_vnode_type()`、`into_vnode_type()` 等转换入口，把 mounted state 与 `VNodeType` 绑在一起。
3. `packages/runtime-vapor/src/runtime/core.rs` 仍导入 `Child`、`VNode`、`VNodeType`，`create_element()` 仍然构造 compat `VNode`。
4. `packages/runtime-vapor/src/runtime/bridge/create_element.rs`、`h.rs` 等桥接文件的注释仍把非重入兜底之外的路径描述为 `vnode_build`。
5. `packages/runtime-vapor/src/runtime/mod.rs` 虽然已不再默认 re-export `VNode / Child / VNodeType`，但整体模块结构仍把 `types.rs` 视作 VDOM 和 MountInput 并存的契约层。

## 重点文件

1. `packages/runtime-vapor/src/runtime/types.rs`
2. `packages/runtime-vapor/src/runtime/core.rs`
3. `packages/runtime-vapor/src/runtime/mod.rs`
4. `packages/runtime-vapor/src/runtime/bridge/create_element.rs`
5. `packages/runtime-vapor/src/runtime/bridge/h.rs`

## 执行清单

- [ ] 删除 `types.rs` 里的 `VNodeType`、`Child`、`VNode` 以及对应转换函数，把剩余需要的类型能力迁到 `MountInputType`、mounted state 或更窄的内部结构。
- [ ] 删除 `core.rs::create_element()` 这条 compat `VNode` 工厂，替换为 `MountInput` builder 或更薄的内部 helper。
- [ ] 清理 `bridge/create_element.rs` 与 `h.rs` 中的 `vnode_build`、`type_tag -> VNode` 心智，统一改成 mount input / render plan 术语。
- [ ] 收口 `types.rs` 顶部注释与整体模块职责，不再把 `VNode / Child / VNodeType` 作为与 `MountInput` 并列保留的协议层。
- [ ] 补一轮 runtime-vapor Rust compile/test，确认没有外部模块仍直接依赖旧类型。

## 阶段边界

1. 不在这一刀里顺手做编译器命名收尾；编译器 vnode IR 放到 [4.md](./4.md)。
2. 不接受“把 `VNode` 移到 legacy/internal.rs 再继续活着”这种折中；这一阶段的目标是仓库删净，而不是换位置隐藏。
3. 如果确实需要临时过渡类型，必须是非递归、非 vnode-shaped 的更窄结构，且不能保留旧命名。

## 完成定义

1. runtime-vapor source 不再定义 `VNode / Child / VNodeType`。
2. `core.rs` 不再有返回 compat `VNode` 的 `create_element()` 工厂。
3. bridge/createElement/h 的默认语义与注释已完全切到 `MountInput` / mounted state。
