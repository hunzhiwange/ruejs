# 清单 5：清空文档、测试、生成产物与最终 grep 例外

真正的“仓库里已经没有虚拟 DOM”，要求的不只是 runtime/source 结构删干净，还包括文档不再把 `VNode` 当当前概念、测试不再把 compat 当保留路径、生成产物不再把旧术语重新写回 repo、grep gate 也不再依赖大段例外名单。

## 目标

1. 清空文档里的当前 VNode / 虚拟 DOM 心智，只在必要的历史迁移说明里保留最小痕迹。
2. 清理测试命名、断言和负向门槛里对 compat / legacy vnode 的显式固化。
3. 刷新 `dist/pkg/pkg-node` 等生成产物，并把最终 grep gate 收缩到尽量无例外的状态。

## 当前阻塞

1. `docs/glossary/index.md` 仍有完整的“虚拟 DOM / VNode”词条，并继续把 Rue 当前主路径解释成保留 `VNode` 公开术语。
2. `docs/guide/reusability/custom-directives.md` 仍把指令 hook 参数写成 `vnode / prevVnode`，并说明它们是底层 `VNode`。
3. `docs/guide/extras/render-function.md` 仍保留 `const vnode = ...`、`vnode.type/props/children/key` 等示例，以及一处 `import type { FC, VNode }` 的残留示例。
4. `packages/runtime/__tests__/renderable-normalize.spec.ts`、`renderable-render-entry.spec.ts`、`teleport.renderable.spec.tsx`、`vaporReactivity.app.spec.tsx` 等仍混杂 compat candidate、compat fragment、stable-compat 命名与断言。
5. `packages/runtime-vapor/tests/runtime_wasm_rue.rs` 等测试注释仍直接写 `vnode`。
6. `packages/runtime/dist/*`、`packages/runtime-vapor/pkg*/*`、`packages/rue-design/dist/*` 等已生成产物仍嵌入 `Legacy VNode/compat`、`vnode` 参数名或过时注释。
7. `scripts/check-compat-cleanup.js` 目前仍依赖多处 allowlist；若要把仓库宣告为无虚拟 DOM 语义，最终 gate 需要尽量收缩例外，甚至新增 repo-level 检查。

## 阶段边界

1. 这一刀不替代前四刀的结构清理；它是仓库级口径与产物收尾。
2. 不把“测试里保留 removed symbol 断言”当作天然允许的例外；如果最终 gate 需要严格 repo grep，就要同步改写测试表达方式。
3. 不在 source 还未稳定前就刷新全部生成产物，避免多次无意义 churn。

## 完成定义

1. 文档中不再把 `VNode / 虚拟 DOM` 当作 Rue 当前主路径的公开心智。
2. 测试与注释不再把 compat / vnode 术语当默认命名。
3. 生成产物已经刷新，仓库级 grep gate 对 `VNode` / `虚拟 DOM` / `__vnode` / `__rue_vnode_id` 的例外已收缩到最小甚至清零。
