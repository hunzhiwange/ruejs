# 清单 4：清理 swc-plugin-rue 中的 vnode IR、helper 命名与编译心智

运行时侧即使已经删干净，只要编译器内部仍把 JSX / slot / 表达式 lowering 统一表述为“转 vnode”，并继续生成 `__vnodeX` 这种 helper 标识符，仓库就仍然保留着虚拟 DOM 作为编译心智。这一刀的目标是把 swc-plugin-rue 的内部协议也从 vnode 语义切走。

## 目标

1. 把编译器内部关于 JSX / slot / fragment / condition lowering 的 vnode 语义改成 renderable / slot value / mount input 语义。
2. 删除 `__vnodeX` 这一类 helper/局部变量命名，避免编译产物与源码继续暴露 vnode 心智。
3. 让编译器测试与注释不再把默认输出描述成“Rue VNode”。

## 当前阻塞

1. `packages/swc-plugin-rue/src/element_expr.rs` 仍写着“统一改写为 Rue VNode 再作为插槽渲染”。
2. `element_fragment.rs` 仍把表达式容器表述为“改写为可渲染 vnode 表达式”。
3. `vapor/block/expr.rs` 仍使用“返回 Rue VNode 的统一形式”“将 JSX 转换为 Rue VNode”等说明。
4. `vapor/helpers.rs` 仍生成 `__vnodeX` 标识符，并在注释中写“统一将任意插槽值规范为 vnode”。
5. `pre/transform.rs`、`vapor/block/mod.rs` 等模块级注释仍把后续阶段描述为“统一转 vnode / 可渲染的 vnode”。

## 重点文件

1. `packages/swc-plugin-rue/src/pre/transform.rs`
2. `packages/swc-plugin-rue/src/element_expr.rs`
3. `packages/swc-plugin-rue/src/element_fragment.rs`
4. `packages/swc-plugin-rue/src/element_slot.rs`
5. `packages/swc-plugin-rue/src/vapor/block/expr.rs`
6. `packages/swc-plugin-rue/src/vapor/block/mod.rs`
7. `packages/swc-plugin-rue/src/vapor/helpers.rs`
8. `packages/swc-plugin-rue/tests/*`

## 执行清单

- [ ] 把表达式、fragment、slot、条件/逻辑 lowering 的 vnode 术语全部改成更贴近当前运行时的 renderable / slot value / mount input 语义。
- [ ] 重命名 `__vnodeX` helper / 局部标识符生成器，避免编译输出继续暴露 vnode 名字。
- [ ] 清理编译器模块注释、测试命名和断言说明中“转 vnode / Rue VNode / vnodeLike”的口径。
- [ ] 用聚焦编译快照确认改名后不会重新产出 `_\$vaporCreateVNode`、`__vnodeX` 或默认 vnode-like helper。

## 阶段边界

1. 这一步优先处理编译器内部协议与命名，不在这里直接刷新全仓所有文档页。
2. 不把“只改注释不改 helper 名字”当作完成；只要 `__vnodeX` 仍在输出，心智就还在。
3. 不允许为了避免 VNode 命名而引入另一套等价的 vnodeLike 命名。

## 完成定义

1. swc-plugin-rue 源码与输出里不再以 vnode 作为默认内部协议术语。
2. `__vnodeX` 与同类 helper 标识符已经消失。
3. 编译器测试不再把默认 JSX/slot lowering 描述为“生成 Rue VNode”。

## 推荐验证

1. `pnpm --filter @rue-js/swc-plugin-rue run build`
2. 相关编译器 tests / 快照回归
3. `rg "\bVNode\b|__vnode|vnodeLike|Rue VNode" packages/swc-plugin-rue`
