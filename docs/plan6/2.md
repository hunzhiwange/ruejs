# 清单 2：拆掉 mounted snapshot / patch / real_dom / lifecycle 的 `VNode` 回退链

仓库里最顽固的虚拟 DOM 残留，不是公开入口，而是 runtime-vapor 的内部回退链：mounted snapshot 仍能 `from_vnode()`，patch 仍能 `into_patch_vnode()`，child 快照和生命周期命名仍围绕 `VNode` 组织。只要这套回路还在，`VNode` 就仍然是默认运行时内部的备用货币。

## 目标

1. 让 mounted snapshot、patch、real_dom 和 lifecycle 只处理 mounted state、`MountInput` 与宿主节点。
2. 删除 `from_vnode()`、`into_patch_vnode()`、`VNode::apply_mounted_subtree()` 这类 mounted state 和 `VNode` 互转链路。
3. 把 child / lifecycle 的剩余 `VNode` 变体和日志命名改成 neutral mounted subtree 语义。

## 当前阻塞

1. `packages/runtime-vapor/src/runtime/types.rs` 仍大量存在 `MountedBlock::from_vnode()`、`MountInput::from_vnode()`、`MountedSubtreeState::from_vnode()`、`MountedComponent::from_vnode()`。
2. 同一文件里仍存在 `into_patch_vnode()`、`MountedSubtreeChild::VNode`、`Child::VNode` 到 snapshot 的映射，以及 `VNode::apply_mounted_subtree()`。
3. `packages/runtime-vapor/src/runtime/render_patch/mod.rs` 及子模块虽然已经以 mounted snapshot 为主，但仍通过这些 `from_vnode()/into_patch_vnode()` 回退 API 保持兼容。
4. `packages/runtime-vapor/src/runtime/real_dom/element.rs`、`fragment.rs` 等仍把子级状态包装成 `MountedSubtreeChild::VNode(...)`。
5. `packages/runtime-vapor/src/runtime/render_lifecycle.rs`、`reactive/core.rs` 仍使用 `runtime:vnode before_unmount`、`VNode before_unmount` 这样的生命周期命名与说明。

## 重点文件

1. `packages/runtime-vapor/src/runtime/types.rs`
2. `packages/runtime-vapor/src/runtime/render_patch/mod.rs`
3. `packages/runtime-vapor/src/runtime/render_patch/children.rs`

8. `packages/runtime-vapor/src/runtime/render_lifecycle.rs`
9. `packages/runtime-vapor/src/reactive/core.rs`

## 执行清单

- [ ] 删除 `types.rs` 中所有 `from_vnode()` / `into_patch_vnode()` / `apply_mounted_subtree()` 回退 API，改成直接从 mounted state 或 `MountInput` 构造目标状态。
- [ ] 把 `MountedSubtreeChild::VNode` 之类的剩余变体改成不带 VDOM 心智的命名，例如更中性的 subtree / node snapshot 变体。
- [ ] 收口 patch 子模块对 `VNode` 回退 API 的依赖，让 same-type update、replace、children diff 只吃 mounted snapshot。
- [ ] 调整 real_dom 子模块对 mounted children 的存储结构，不再用 `VNode` 名称包装 snapshot。
- [ ] 清理 lifecycle / reactive 模块里的 `VNode before_unmount` 命名、日志标签和说明，统一改成 mount lifecycle / subtree lifecycle。

## 阶段边界

1. 这一步先拆回退链，不要求同一刀就把 `VNode / Child / VNodeType` 结构定义全部删掉。
2. 不允许把 `VNode` 只换个 enum variant 名字而保留相同的 snapshot-to-tree 回退机制。
3. 如果某些极少数测试仍需旧结构，只能迁到专门测试辅助，不能继续占据默认 runtime 路径。

## 完成定义

1. mounted snapshot / patch / real_dom / lifecycle 主路径不再调用任何 `from_vnode()` 或 `into_patch_vnode()`。
2. `MountedSubtreeChild::VNode`、`Child::VNode` 这类默认 runtime 变体已经移除或改成无 VDOM 含义的名字和结构。
3. 生命周期与 reactive 说明里不再把卸载/cleanup 过程表述为 `VNode` 递归。
