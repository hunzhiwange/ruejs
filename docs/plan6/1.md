# 清单 1：删掉默认 JS / Wasm 边界上的 vnode-like 输入协议

第五阶段已经让默认主入口拒绝 legacy `VNode` / compat 输入，但“拒绝”不等于“协议已经消失”。只要默认 JS runtime 还保留 `kind: 'compat'` 分支，Wasm / real_dom 边界还继续识别 `type / props / children / key` 这类 dev object 形状，仓库里就仍然保留着一套活跃的 vnode-like 输入协议。

## 目标

1. 让 `packages/runtime` 的默认 normalize 逻辑变成 renderable-only，不再返回 `kind: 'compat'`。
2. 让 runtime-vapor 的默认 bridge / convert 路径不再接受 vnode-like dev object，也不再把 `MountInput` 回转成 dev object。
3. 把默认入口的错误、日志与测试口径从 `Legacy VNode/compat` 改成“默认路径不接受对象协议”。

## 当前阻塞

1. `packages/runtime/src/renderable.ts` 仍定义 `NormalizeRenderableResult = { kind: 'renderable' } | { kind: 'compat' }`。
2. `packages/runtime/src/renderable-normalize.ts` 仍把 plain object 与混合数组保留为 compat candidate，而不是直接拒绝。
3. `packages/runtime/__tests__/renderable-normalize.spec.ts` 仍在断言“plain object inputs as compat candidates”。
4. `packages/runtime-vapor/src/runtime/bridge/input.rs` 里的 `looks_like_dev_object()`、`mount_input_from_dev_object_or_registry()`、`default_mount_input_from_input()` 仍会把默认输入解析为 dev object / registry / vnode-like shape。
5. `packages/runtime-vapor/src/runtime/real_dom/convert.rs` 仍保留 `looks_like_default_vnode_object()`、`dev_object_to_input()`、`input_to_dev_object()`，继续维护默认 dev object 协议。
6. `packages/runtime-vapor/src/runtime/bridge/render.rs`、`render_anchor.rs`、`render_between.rs`、`render_static.rs` 的注释和 warning 仍写成“接受对象/compat 输入，解析为 VNode”。

## 执行清单

- [ ] 删除 `NormalizeRenderableResult.kind = 'compat'` 分支，把默认 normalize 改成 renderable-only 或显式 error 结果。
- [ ] 重写 `renderable-normalize.spec.ts` 与相关 runtime tests，不再把 plain object/vnodeLike 当作“保留的 compat candidate”。
- [ ] 调整 `packages/runtime/src/rue.ts` 的默认输入分析逻辑，去掉对 compat 分支的依赖，让默认主路径只接受 renderable、mount handle、raw node、block 与合法数组。
- [ ] 删除 `bridge/input.rs` 默认路径上的 dev object 识别逻辑，保留的 registry handle / raw node / function component 解析必须是非 vnode-like 协议。
- [ ] 删除或封存 `real_dom/convert.rs` 中默认 `input_to_dev_object()` / `dev_object_to_input()` 互转回路，不再让默认主路径生产或消费 vnode-like dev object。
- [ ] 把默认入口错误文案、日志标签和测试名称从 `Legacy VNode/compat` 收口到更中性的“unsupported object input / removed legacy object protocol”。

## 阶段边界

1. 这一刀优先清默认边界协议，不在这里直接删除 runtime-vapor 内核里的 `VNode` 结构体。
2. 不允许用“默认入口直接抛错，但内部继续保留 compat shape”作为完成；协议本身必须删掉。
3. 如果仍需某些 dev object 辅助，只能留在专用测试工具或显式内部模块，不能继续出现在默认路径。

## 完成定义

1. `packages/runtime` 默认 normalize 逻辑不再出现 `kind: 'compat'`。
2. runtime-vapor 默认 `default_mount_input_from_input()` 不再识别 vnode-like dev object。
3. `real_dom/convert.rs` 默认路径不再有 `dev_object_to_input()` / `input_to_dev_object()` 这种对象协议互转。
4. 默认入口错误、日志与测试不再把 legacy object 协议命名为 `VNode/compat`。
