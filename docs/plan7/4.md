# 清单 4：实现调用侧降级，把 `<Foo />` 消费链路切到原生标签协议

如果定义侧已经能产出 custom element 构造器，但调用侧 `<Foo />` 仍然被 swc 编译成 Rue 组件分支和 `renderAnchor()`，那就还不算“全部原生组件改造”。真正的分水岭在于：opt-in 组件的消费协议也要切成原生 custom element 标签、宿主属性、slot 和事件监听，而不是继续走 Rue 组件协议。

## 目标

1. 让 swc 在消费侧识别“这是一个 custom element 化的组件符号”，并走 native element 生成路径。
2. 把组件 props、事件、slot、ref 和 key 等语义下沉到宿主节点协议，而不是 `renderAnchor()`。
3. 为同模块和跨模块消费建立 `Symbol -> tagName` 的映射来源。

## 当前阻塞

1. `packages/swc-plugin-rue/src/utils.rs` 的组件识别规则仍是“大写开头就是组件”，没有“custom element 组件”这个中间态。
2. `packages/swc-plugin-rue/src/elements.rs` 遇到组件后直接进入 `element_component.rs` 分支。
3. `packages/swc-plugin-rue/src/element_component.rs` 当前会把 children 重写到 `props.children`，并最终走 `renderAnchor()`。
4. 当前没有 `Ident -> tagName` 的追踪表，也没有跨模块导入后的 tag 映射策略。
5. 当前没有 native custom element 专用的 props lowering：哪些落 attribute、哪些落宿主属性、哪些落 `el.props`。
6. 当前没有 native custom element 专用的事件 lowering：`onSave` / `onChange` 该怎么映射到 `addEventListener()` 或宿主属性。

## 重点文件

1. `packages/swc-plugin-rue/src/utils.rs`
2. `packages/swc-plugin-rue/src/elements.rs`
3. `packages/swc-plugin-rue/src/element_component.rs`
4. `packages/swc-plugin-rue/src/element_children.rs`
5. `packages/swc-plugin-rue/src/element_slot.rs`
6. `packages/swc-plugin-rue/src/attrs.rs`
7. `packages/swc-plugin-rue/src/vapor/component.rs`

## 执行清单

- [ ] 在 transform 上下文中维护“哪些组件符号已经是 custom element 组件”以及它们对应的 `tagName`。
- [ ] 修改组件分支判定：对 native custom element 组件走原生元素生成路径，而不是 `renderAnchor()`。
- [ ] 设计 props lowering：
  - 可序列化标量优先落 attribute；
  - 复杂值落宿主属性或 `el.props`；
  - 删除值时同步删除 attribute 或回写 `undefined`。
- [ ] 设计事件 lowering：将消费侧事件处理绑定到宿主 `addEventListener()`，并对 `CustomEvent.detail` 结构做统一约定。
- [ ] 设计 children / slot lowering：
  - 默认 children 对应默认 slot；
  - 命名 slot 需要有生成规则；
  - 不能再默认改成 `props.children`。
- [ ] 校准 `ref`、`key`、列表复用和更新路径，确保生成的宿主节点仍符合现有 vapor 更新模型。
- [ ] 为同模块与跨模块消费增加 fixtures，至少覆盖：无 props、标量 props、复杂 props、命名 slot、宿主事件监听。

## 阶段边界

1. 不把框架控制组件强塞进这套 native 标签 lowering；它们应依赖 [5.md](./5.md) 的白名单判定。
2. 不接受“native custom element 组件仍偷偷走 `renderAnchor()`”这种折中；消费协议必须真正变成原生标签。
3. 如果某类 props / children 语义无法自然映射到 native custom element，就必须显式列入限制，而不是静默降级成错误行为。

## 完成定义

1. opt-in 组件在消费侧已经走原生 custom element 标签和宿主协议。
2. `element_component.rs` 不再是这些组件的主路径，`renderAnchor()` 只保留给仍属于 Rue 组件协议的组件。
3. 组件 props、slot、事件和 ref 的 lowering 规则已经固定并有测试覆盖。

## 推荐验证

2. `cargo test -p swc_plugin_rue slot`
3. 新增浏览器 / jsdom fixture，验证宿主属性、事件和 slot 投影
