# 社区指南

Rue 社区正在飞速发展，如果你正在阅读本文，很可能已经准备好加入我们了。那么欢迎你！

现在，我们将为你解答社区能为你提供什么，以及你能为社区做些什么。

## 资源

### 行为准则

我们的[行为准则](/#/page/about/community-guide)旨在帮助所有成员和我们所参与的技术社区共同进步、更加丰富。

### 获取支持

- https://github.com/ruejs：如果你发现了 bug 或有新功能建议，欢迎在 GitHub issue 中提交。请遵守各仓库 issue 模板中注明的规则。

## 你能做什么

### 帮助其他用户

代码贡献并不是参与 Rue 社区的唯一方式。在 Discord 或论坛上回答其他 Rue 用户的问题，同样是非常有价值的贡献。

### 协助分类 Issue

Issue 分类意味着收集缺失的信息、复现问题、验证问题的有效性，并调查问题产生的原因。

我们每天都会在 https://github.com/ruejs 收到大量 issue。相对于用户规模，我们的精力有限，仅 issue 分类就需要团队投入大量时间。通过协助我们分类 issue，你就是在帮助我们提高效率，让我们能够将时间用于更高优先级的工作。

你不一定要以修复 issue 为目标来进行分类（当然能修复就更好了）。例如，分享你调查的结果（比如导致 bug 的提交记录），就已经能为我们节省大量时间。

### 贡献代码

贡献 bug 修复或新功能，是你能做出的最直接的贡献形式。

Rue 核心仓库提供了https://github.com/ruejs/core/blob/main/.github/contributing.md，其中包含了 Pull Request 指南、构建设置和高层架构相关的信息。其他子项目仓库也可能包含自己的贡献指南——请在提交 Pull Request 前务必阅读。

我们随时欢迎 bug 修复。对于新功能，建议先在 https://github.com/ruejs/rfcs/discussions 中讨论使用场景和实现细节。

### 分享（并积累）你的经验

除了在论坛和聊天室回答问题、分享资源之外，还有一些不那么明显的方式可以分享和扩展你的知识：

- **编写学习材料。** 常说最好的学习方法就是去教授。如果你在使用 Rue 做一些有趣的事情，可以通过写博客、开设 workshop，甚至在社交媒体上分享代码片段来巩固你的专业知识。
- **关注你关心的仓库。** 这会在该仓库有动态时向你发送通知，让你了解正在进行的讨论和即将推出的新功能。这是积累专业知识的绝佳方式，最终你将能够帮助处理 issue 和 Pull Request。
