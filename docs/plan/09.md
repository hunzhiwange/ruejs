# 09：迁移 useComponent 与 Teleport

## 目标

优先迁移两个容易暴露协议缺口的 built-ins：异步组件与跨容器渲染。

## 只做这些事

1. useComponent 不再通过 vnodeLike + render 驱动容器。
2. Teleport 直接消费 Renderable / Block。
3. 验证 loading、error、resolved 和目标容器切换行为。

## 建议涉及文件

1. packages/runtime/src/hooks/useComponent.ts
2. packages/runtime/src/components/Teleport.ts

## 不要顺手做

1. 不改 Transition。
2. 不切公开类型。
3. 不动 compat 导出策略。

## 完成标准

1. useComponent 和 Teleport 主实现不再主动创建 VNode。
2. 异步组件三态和 Teleport 目标切换不回归。
3. 相关测试和示例通过。

## 回退点

如果 Teleport 的锚点迁移风险太大，先只切渲染协议，保留旧的块搬运方式。
