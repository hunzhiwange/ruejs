# 第二阶段：迁移编译器和主路径调用方

这一阶段开始让默认应用路径真正跑在新协议上，但 compat 仍然保留。

## 阶段目标

1. swc-plugin-rue 不再默认产出 vnodeLike 思维。
2. Router 和 built-ins 主实现改为消费 Renderable / Block。
3. 默认应用热路径尽量不再依赖 h 和 VNode。

## 微阶段清单

1. [06.md](./06.md)：编译器 imports 与 slot / children 规范化
2. [07.md](./07.md)：编译器组件、列表、表达式容器改造
3. [08.md](./08.md)：迁移 Router
4. [09.md](./09.md)：迁移 useComponent 与 Teleport
5. [10.md](./10.md)：迁移 Transition 与 TransitionGroup

## 阶段边界

这一阶段不要做以下事情：

1. 不删除 compat VNode。
2. 不把公开类型一次性切成 Renderable-only。
3. 不在没有验证的情况下同时改编译器和所有内置模块。

## 阶段完成定义

当以下条件成立时，第二阶段结束：

1. 编译器主路径不再依赖 \_$vaporCreateVNode。
2. Router 与核心 built-ins 不再主动构造 vnodeLike。
3. 主要示例和站点页面跑在新协议上。
4. compat 仍然可用于手写 h 和渲染函数回退。
