# 02：实现 normalizeRenderable

## 目标

建立统一的输入规范化入口，把不同形态的 children / 返回值收敛到新协议上。

## 只做这些事

1. 新增 normalizeRenderable。
2. 先支持字符串、数字、DOM 节点、DocumentFragment、数组、Block。
3. 为后续 compat VNode 预留分支，但本步不要求完成 compat 适配。

## 建议涉及文件

1. packages/runtime/src/renderable-normalize.ts
2. packages/runtime/src/renderable.ts
3. 必要的最小测试文件

## 不要顺手做

1. 不修改 render / renderBetween 入口。
2. 不迁移 vaporCreateVNode 调用点。
3. 不引入 Router 或 built-ins 改动。

## 完成标准

1. normalizeRenderable 可独立测试。
2. 数组和文本输入的规范化规则固定下来。
3. pnpm check 与对应测试通过。

## 回退点

如果数组或 DOM 语义拿不准，先支持最小集合，不要提前兼容所有历史输入。
