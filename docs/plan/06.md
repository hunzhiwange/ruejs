# 06：编译器 imports 与 slot / children 规范化

## 目标

先把编译器最容易收敛的输入口改掉：imports、slot 和 children 规范化。

## 只做这些事

1. 调整 swc-plugin-rue 的运行时导入，改为依赖新协议入口。
2. 替换 slot / children 中对 \_$vaporCreateVNode 和 h('fragment', ...) 的依赖。
3. 只覆盖 slot / children，不碰组件边界和列表主逻辑。

## 建议涉及文件

1. packages/swc-plugin-rue/src/imports.rs
2. packages/swc-plugin-rue/src/element_slot.rs
3. 必要时补 children 相关帮助模块

## 不要顺手做

1. 不改 Router。
2. 不改 Teleport / Transition。
3. 不一次性处理 component、list、expr container。

## 完成标准

1. 编译输出中 slot 路径不再默认依赖 \_$vaporCreateVNode。
2. slot / children 编译快照稳定。
3. swc-plugin-rue 的 test 和 check 通过。

## 回退点

如果 children 规范化范围太大，先只处理 slot，children 继续走兼容路径。
