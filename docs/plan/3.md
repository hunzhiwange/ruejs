# 第三阶段：切默认、收 compat、更新文档

这一阶段才处理公开类型、compat 收缩和文档/发布。

## 阶段目标

1. 把公开类型与默认出口切到 Renderable / Block。
2. 决定 compat 是保留还是继续收缩。
3. 更新文档、升级说明和发布策略。

## 微阶段清单

1. [11.md](./11.md)：切换 JSX runtime 与公开类型桥接
2. [12.md](./12.md)：抽 compat、切默认出口
3. [13.md](./13.md)：更新文档、发布策略与迁移说明

## 决策点

进入这一阶段前，需要明确二选一：

1. 保留 compat 子路径一段时间。
2. 在 major 版本中直接移除主路径 VNode。

如果没有这个决策，不要开始 12.md。

## 阶段完成定义

当以下条件成立时，第三阶段结束：

1. 默认公开语义已经不是 VNode。
2. compat 策略已经明确并落地。
3. 文档与升级说明已经同步完成。
4. 仓库中的 VNode 已经只剩 compat 子路径，或者已彻底删除。
