# 05：建立 compat VNode 适配器

## 目标

把旧 VNode 正式收缩为 compat 路径，而不是继续和新协议混在一起。

## 只做这些事

1. 新增 VNode 到 Renderable 的 compat adapter。
2. 让旧 h / createElement 产物通过该 adapter 进入新入口。
3. 不改变业务层旧调用方式。

## 建议涉及文件

1. packages/runtime/src/renderable-compat.ts
2. packages/runtime/src/rue.ts
3. packages/runtime-vapor/src/runtime/bridge/create_element.rs

## 不要顺手做

1. 不改 swc-plugin-rue。
2. 不切公开类型。
3. 不删除旧的 createElement / patch 逻辑。

## 完成标准

1. compat 边界清晰可见。
2. 新协议与旧 VNode 混用时仍然稳定。
3. 第一阶段结束后，后续模块迁移不再直接依赖裸 VNode 入口。

## 回退点

如果适配器实现复杂，先保证“可桥接”，不要在这一步强行清理所有旧逻辑。
