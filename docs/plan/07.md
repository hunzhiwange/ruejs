# 07：编译器组件、列表与表达式容器改造

## 目标

把编译器中剩余的 vnodeLike 热路径拆掉，尤其是组件边界、列表和表达式容器。

## 只做这些事

1. 迁移组件边界输出到 Renderable / Block。
2. 迁移列表与表达式容器，减少 renderBetween(vnodeLike, ...) 生成。
3. 保持旧组件返回 VNode 时仍可走 compat。

## 建议涉及文件

1. packages/swc-plugin-rue/src/element_component.rs
2. packages/swc-plugin-rue/src/element_list.rs
3. packages/swc-plugin-rue/src/vapor/component.rs
4. packages/swc-plugin-rue/src/vapor/block/expr_container.rs

## 不要顺手做

1. 不改 built-ins。
2. 不切 jsx-runtime 公开语义。
3. 不删除 compat 回退。

## 完成标准

1. 组件、列表、表达式容器主输出不再依赖 vnodeLike。
2. 编译快照和相关示例通过。
3. 对旧组件返回值仍有回退路径。

## 回退点

如果列表改造风险最大，可先迁移组件和表达式容器，再单独补列表。
