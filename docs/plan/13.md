# 13：更新文档、发布策略与迁移说明

## 目标

让仓库文档、升级说明和发布节奏与新默认渲染机制保持一致。

## 只做这些事

1. 更新渲染机制和 render-function 文档。
2. 更新 getting-started、intro 以及相关 built-ins 文档。
3. 写清 compat 的适用范围、废弃计划或移除说明。
4. 补充升级指南和 release note 方向。

## 建议涉及文件

1. docs/guide/extras/rendering-mechanism.md
2. docs/guide/extras/render-function.md
3. docs/getting-started.md
4. docs/intro.md
5. 必要的发布说明或升级指南文件

## 不要顺手做

1. 不再回头改运行时主逻辑。
2. 不把新增技术债塞回计划里。
3. 不在文档中保留旧结论和新结论并存的模糊说法。

## 完成标准

1. 文档已经明确默认机制是 Block / Vapor，而不是 VNode。
2. compat 策略写清楚了。
3. 升级说明足以指导旧用户迁移。

## 回退点

如果默认出口还没最终定案，先把文档写成“新默认方向 + compat 过渡期”，不要过早宣称 VNode 已完全移除。
