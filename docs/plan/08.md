# 08：迁移 Router

## 目标

先把应用主路径中的 RouterView 和 RouterLink 从 VNode 习惯中移出来。

## 只做这些事

1. RouterView 直接挂载 Renderable。
2. RouterLink 直接渲染锚点块，不再依赖 h('a', ...)。
3. 路由切换时由 BlockInstance 承担旧内容清理。

## 建议涉及文件

1. packages/router/src/index.ts
2. 必要的 router 测试文件

## 不要顺手做

1. 不改 Teleport。
2. 不改 Transition。
3. 不处理 jsx-runtime。

## 完成标准

1. Router 不再主动构造 vnodeLike。
2. 路由切换和链接点击行为不回归。
3. 使用路由的站点页面可正常构建和运行。

## 回退点

如果 RouterLink 改造牵连过大，可先迁移 RouterView，再单独处理 RouterLink。
