# 03：桥接 render 与 renderBetween

## 目标

让现有渲染入口可以识别 Renderable，但仍然兼容旧 VNode 路径。

## 只做这些事

1. 在 render / renderBetween 中接入 normalizeRenderable。
2. 新协议走新入口，旧 VNode 继续走旧桥接。
3. 保持业务层 API 不变。

## 建议涉及文件

1. packages/runtime/src/rue.ts
2. packages/runtime/src/index.ts
3. packages/runtime-vapor/src/runtime/bridge/render.rs

## 不要顺手做

1. 不做 Block 清理机制重构。
2. 不改 createElement / h。
3. 不改编译器输出。

## 完成标准

1. render 和 renderBetween 可消费 Renderable。
2. 旧 VNode 调用不回归。
3. 最小挂载与区间渲染测试通过。

## 回退点

如果 Rust 入口改动过大，可以先在 TS 层桥接，再把 Rust 适配留到下一步。
