# 04：引入 Block 清理拥有者

## 目标

把副作用、事件和 effect scope 的清理责任，从 VNode before_unmount 抽到 BlockInstance 自身。

## 只做这些事

1. 为 BlockInstance 增加 cleanup bucket 或等价机制。
2. 支持在 unmount 时统一释放资源。
3. 为后续 Router 和 built-ins 迁移打基础。

## 建议涉及文件

1. packages/runtime/src/renderable-lifecycle.ts
2. packages/runtime-vapor/src/runtime/render_lifecycle.rs
3. 必要的响应式清理测试

## 不要顺手做

1. 不迁移 built-ins。
2. 不抽 compat adapter。
3. 不修改编译器。

## 完成标准

1. Block 卸载有独立清理路径。
2. 重复 mount / unmount 不会累积 effect。
3. 对现有 VNode 生命周期不造成行为破坏。

## 回退点

如果一次性完全替换风险过大，允许先做并行清理机制，再逐步切换主路径。
