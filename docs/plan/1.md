# 第一阶段：建立协议，不切默认路径

这一阶段只做底座，不迁移编译器和内置组件主路径。

## 阶段目标

1. 定义 Renderable / Block 基础协议。
2. 让 render、renderBetween 能识别新协议。
3. 把清理责任从 VNode 生命周期中抽离。
4. 为旧 VNode 提供 compat 适配层。

## 微阶段清单

1. [01.md](./01.md)：定义 Renderable 核心类型
2. [02.md](./02.md)：实现 normalizeRenderable
3. [03.md](./03.md)：桥接 render / renderBetween
4. [04.md](./04.md)：引入 Block 清理拥有者
5. [05.md](./05.md)：建立 compat VNode 适配器

## 阶段边界

这一阶段不要做以下事情：

1. 不改 swc-plugin-rue 输出结构。
2. 不迁移 Router、Teleport、Transition 等 built-ins。
3. 不删除 h、createElement、patch。
4. 不切换默认公开类型。

## 阶段完成定义

当以下条件成立时，第一阶段结束：

1. 新协议已经存在并可被导出。
2. render / renderBetween 可以识别新协议和 compat VNode。
3. Block 的卸载清理不再完全依赖 VNode before_unmount。
4. 旧代码仍然能跑，业务层 API 暂不破坏。
